# 법적 준수 사항 (Legal Compliance)

**작성일:** 2026년 2월 9일
**기준 법률:** 위치정보의 보호 및 이용 등에 관한 법률, 개인정보 보호법

---

## 📌 개요

챙겨(Chaeng-gyeo) 앱은 사용자의 GPS 좌표를 수집·이용하여 위치 기반 날씨 알림 서비스를 제공합니다. 이에 따라 한국의 **위치정보법** 및 **개인정보보호법**을 준수해야 합니다.

---

## 🚨 주요 법적 의무 사항

### 1. 위치기반서비스 사업자 신고 의무 ⭐ (최우선)

**법적 근거:**
- 위치정보법 제5조 (위치기반서비스사업의 허가 등)

**요구사항:**
- 방송통신위원회에 **위치기반서비스사업자 신고** 필수
- GPS 좌표(위도/경도)를 수집·이용하는 모든 서비스 해당
- 서버에 저장하지 않고 실시간 조회만 하더라도 신고 대상

**신고 절차:**
1. [위치정보지원센터](https://www.lbsc.kr/) 접속
2. 위치기반서비스사업 신고서 제출
3. 필요 서류:
   - 사업계획서
   - 개인위치정보 보호약관
   - 위치정보관리책임자 지정 서류
4. 비용: 무료

**예외 규정:**
- 소상공인 또는 1인 창조기업: 사업 개시 후 **1개월 내 신고 가능**

**미신고 시 처벌:**
- 3년 이하 징역 또는 3천만원 이하 벌금

---

### 2. 사전 동의 획득 의무

**법적 근거:**
- 위치정보법 제15조 (개인위치정보의 수집 등의 금지)
- 개인정보보호법 제15조 (개인정보의 수집·이용)

**요구사항:**
- 개인위치정보 수집 전 **명시적 동의** 필수
- OS 위치 권한(Location Permission) ≠ 법적 동의
- 별도의 동의 화면 및 약관 제공 필요

**동의서에 포함해야 할 내용:**
```
✓ 수집·이용 목적: 사용자 위치 기반 날씨 정보 제공 및 알림
✓ 수집 항목: GPS 좌표 (위도, 경도)
✓ 보유 및 이용 기간:
  - Option A: 즉시 파기 (알림 발송 시점에만 사용)
  - Option B: 회원 탈퇴 시까지 (별도 동의 시)
✓ 동의 거부 권리 및 불이익: 동의하지 않을 경우 서비스 이용 불가
✓ 제3자 제공 여부: 없음 (기상청 API는 좌표 기반 조회만 수행)
```

**미동의 시 처벌:**
- 3년 이하 징역 또는 3천만원 이하 벌금

---

### 3. 확인자료 보관 의무 (6개월)

**법적 근거:**
- 위치정보법 제16조 (개인위치정보주체의 권리 등)
- 위치정보법 시행령 제14조

**요구사항:**
- 위치정보 수집·이용·제공 사실을 **자동으로 기록**
- 기록은 **6개월 이상 보관** 필수
- 사용자 요청 시 열람 가능하도록 제공

**구현 방법:**
```javascript
// Firestore 로깅 컬렉션 예시
logs/{userId}/locationAccessLogs/{logId}
{
  timestamp: "2026-02-09T07:00:00Z",
  action: "collected" | "used" | "provided",
  purpose: "weather_notification",
  location: { latitude: 37.5665, longitude: 126.9780 },
  deviceInfo: "iOS 18.0",
  ipAddress: "123.456.789.000" (선택)
}
```

**사용자 제공 기능:**
- 설정 화면 → "위치정보 이용 내역" 메뉴 추가
- 최근 6개월 기록 표시

---

### 4. 개인위치정보 파기 의무

**법적 근거:**
- 위치정보법 제16조의2 (개인위치정보의 파기)

**원칙:**
- 수집·이용·제공 목적을 달성한 때 **즉시 파기**
- 확인자료(로그)는 6개월 보관 후 파기

**적용 방안:**

#### Option A: 서버 미저장 방식 (권장 ⭐)
```
[Cloud Function 알림 발송 시]
1. 사용자 기기에 위치 조회 요청 (Push)
2. 기기가 현재 위치 전송
3. Cloud Function이 즉시 기상청 API 호출
4. 알림 발송 후 위치 정보 즉시 파기
5. 로그만 6개월 보관
```

**장점:**
- 파기 의무 자동 준수
- 데이터 유출 위험 최소화
- 법적 리스크 감소

#### Option B: 별도 동의 기반 보관
```
- "위치 정보 저장" 항목을 별도 동의 항목으로 분리
- 동의 시: Firestore에 보관 (탈퇴 시 즉시 삭제)
- 미동의 시: 실시간 조회만 사용
```

**회원 탈퇴 시:**
- 모든 개인위치정보 즉시 삭제
- 로그는 익명화 후 6개월 보관

---

### 5. 개인정보처리방침 공개 의무

**법적 근거:**
- 위치정보법 제18조 (위치정보의 보호 조치)
- 개인정보보호법 제30조 (개인정보 처리방침의 수립 및 공개)

**공개 위치:**
- 앱 내 설정 화면
- 회사 홈페이지 (있을 경우)
- 앱스토어 설명

**필수 포함 내용:**
```
1. 위치정보 처리 목적
   - 사용자 위치 기반 날씨 정보 제공
   - 날씨 알림 서비스 제공

2. 수집 항목 및 방법
   - GPS 좌표 (위도, 경도)
   - 자동 수집 (사용자 기기의 위치 서비스)

3. 보유 및 이용 기간
   - 즉시 파기 또는 회원 탈퇴 시까지

4. 제3자 제공 여부
   - 제공하지 않음

5. 확인자료 보관 기간
   - 6개월

6. 파기 절차 및 방법
   - 전자적 파일: 복구 불가능한 방법으로 영구 삭제
   - 회원 탈퇴 시 즉시 삭제

7. 위치정보관리책임자
   - 성명: [이준현]
   - 연락처: [이메일/전화번호]
   - 소속: [챙겨 개발팀]

8. 권리 및 행사 방법
   - 동의 철회 권리
   - 위치정보 이용·제공 사실 확인 자료 열람 요구권
   - 손해배상 청구권
```

## 💡 권장 아키텍처 (법적 리스크 최소화)

### 현재 계획 (Firestore에 위치 저장)
```
[문제점]
- 위치 정보 장기 보관으로 파기 의무 위반 가능성
- 데이터 유출 시 법적 책임 증가
```

### 권장 변경안 (서버 미저장)
```
1. Firestore 저장 항목
   users/{userId}/settings/
   - notificationTime: "07:00"
   - notifications: { rain: {...}, dust: {...} }
   - fcmToken: "..."
   ❌ location 필드 제거

2. 알림 발송 플로우
   [Cloud Function - 07:00 실행]
   ↓
   [FCM으로 "위치 조회 요청" 푸시]
   ↓
   [사용자 앱이 현재 위치 획득]
   ↓
   [앱이 직접 기상청 API 호출]
   ↓
   [조건 충족 시 로컬 알림 표시]
   ↓
   [위치 정보 즉시 파기]

3. 로그만 서버에 저장
   logs/{userId}/locationAccessLogs/{logId}
   - timestamp
   - action: "used"
   - purpose: "weather_check"
```

**장점:**
- ✅ 위치 정보 즉시 파기 자동 준수
- ✅ 개인정보 유출 위험 최소화
- ✅ 법적 분쟁 시 유리
- ✅ GDPR 등 국제 규정에도 부합
